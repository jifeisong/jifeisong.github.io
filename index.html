StructStreaming

发表于 2017-06-19 | 分类于 spark |

基于spark2.1.1的文档翻译

Overview

structured streaming是一种基于Spark SQL引擎构建的可扩展且容错的流处理引擎。 您可以以静态数据表示批量计算的方式来表达流式计算。 Spark SQL引擎将随着流式传输数据持续到达而逐渐持续运行，并更新最终结果。 您可以使用Scala，Java或Python中的Dataset / DataFrame API来表达流聚合（Streaming aggregation），事件时间窗口（event-time windows），流到批处理连接（stream-to-batch）等。计算在相同的优化的Spark SQL引擎上执行。 最后，系统通过检查点和预写日志（wal）确保端到端的一次容错（end-to-end exactly-once fault-tolerance）保证。 简而言之，structured streaming提供快速，可扩展，容错，端到端的一次（end-to-end exactly-once）流处理，而无需用户理解streamming。
Structured Streaming 在spark2.1版本仍然是测试版，我们将引导您完成学习编程模型和API。首先以一个简单的例子开始-a streaming word count。

SparkStreaming-容错处理

发表于 2017-06-13 | 分类于 spark |

背景

为了理解 Spark Streaming 提供的语义，让我们记住 Spark RDDs 最基本的容错语义。

  • RDD 是不可变的，确定性的可重新计算的分布式的 DataSet（数据集）。每个RDD都拥有它的所有的依赖信息，而利用这些信息就可以从错误中恢复。
  • 如果任何分区的 RDD 由于 Worker 节点失败而丢失，那么这个分区能够根据依赖关系从原始数据集中恢复出来。
  • 假设所有的 RDD 转换是确定的，在最终转换的 RDD 中的数据总是相同的，与Spark 群集中的失败无关。

spark-共享变量

发表于 2017-06-09 | 分类于 spark |

通常情况下，一个传递给 Spark 操作（例如 map 或 reduce）的函数 func 是在远程的集群节点上执行的。该函数 func 在多个节点执行过程中使用的变量，是同一个变量的多个副本。这些变量的以副本的方式拷贝到每个机器上，并且各个远程机器上变量的更新并不会传播回 driver program（驱动程序）。通用且支持 read-write（读-写） 的共享变量在任务间是不能胜任的。所以，Spark 提供了两种特定类型的共享变量 : broadcast variables（广播变量）和 accumulators（累加器）。
